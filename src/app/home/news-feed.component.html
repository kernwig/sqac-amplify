I called a guest tip last Tuesday and tried to use SqAC for it. Boy was I embarrassed when it seemed like the same modules were being used over and over... and after I thought I just fixed that! This weekendÔ∏è I took another look at the code and (this is nerdy, feel free to skip):

  • Swapped in an improved pseudo-random number generator that accounts for binary bias.
  • Replaced the scattershot approach to finding the best next module with a targeted and consistent algorithm.

While at it, I made some related enhancements and found and fixed a couple other problems:

  • Added some details about how each module is chosen into the tip history display, to provide a clue the next time the selection quality seems poor. Also added display of start and end formation to the module name displayed above the call list, because it's useful to see!
  • Sometimes session restrictions weren't initially applied to the module selector; it is now.
  • Ignores the impossible should you select of average difficulty greater than max difficulty; which would certainly skew the selections.

Finally, I split the Circulate family into Circulate B1 and Circulate B2. Even though there's really only one Circulate family, the CALLERLAB teaching order calls for teaching some in Basic 1 and some later in Basic 2. Also, community dance programs tend to only teach the simple (Basic 1) calls. This allows you to select only Circulate B1 for your dance session.

This release includes a hodge podge of enhancements and bug fixes to make SqAC better still.

  • There was a feature to avoid using the same module twice in one tip. This never seemed to work though, because it didn't. Now it does.
  • When naming a module, a little warning tells you in the name is already used by another module. You can have the same name if you want, but I sure found this helpful when entering 300+ new basic modules from Ceder.net.
  • The Alternative Command for a call in a module may now start with text like (Heads/Sides) or (Side/Head), and the when danced it'll now automatically choose one word at random and display that. This allows a single get-in module to work two different ways.
  • I was curious about the distribution of the 383 modules now in the Ceder.net Basic collection, so now after all the list of modules in a collection there's a dandy little table called Summary of Start & End Formations.

Did I mention that there are now three times as many modules in the Ceder.net Basic Selections collection? üéâ

Offline support has been coming incrementally. I'm pleased to report that as of today's release, you can even edit your collections offline. You're device's storage is now the primary storage, with the cloud being used for backup and synchronization across devices.

Ah, but now the cloud storage does a bit more for you too. The last several revisions of your collections are now preserved on the cloud. You can now click the history icon to see these older revisions and revert back to them if you made a mistake. Cool, huh?

A Progressive Web App is a application that uses modern web capabilities to deliver a native-like experience for users, while still working as an ordinary web app on older systems. SqAC is now a real Progressive Web App (PWA)!

What does this mean?

  • Visit SqAC a couple of times, and your browser should offer to install it. Exactly how this looks depends on your operating system. Screenshots .
  • You can run SqAC again while offline, and it'll just work! (Saving changes while offline will come soon.)

The bad news is, only Chrome and Firefox fully supports PWA right now. Microsoft and Apple are working on it though, with support expected in 2018.

There's a whole new section of the app called Sessions. With it you can track multiple dancer audiences for classes or club dances. Each session not only lets you toggle dance level, BPM, and difficulty for each of your regular classes and dances, but also:

  • Limit which collections you want to pull modules from. After all, you probably don't want to use your teaching modules at a club dance!
  • Set which call families your dancers know. This is particularly useful at a class or unusual dance level. (I'm looking at you, Club 50!)

Added a toggle switch when editing a collection (name, description, license), so that you can now designate your collection as public or private.

I was annoyed when calling a tip and SqAC did the same module twice in a row! It'll try not to do that anymore. (It still can, if the choices are limited.)

Module editor enhancements: the call selection popup is now wider, and estimated dance time now shown under the call sequence.

Finally, I took a couple steps toward making SqAC work offline. You probably won't notice them yet, but they are steps toward that goal.

You can now Play (dance) modules on-demand from in their collection. This is great for teach modules! Search online for CALLERLAB SSD for the collections I'm building up based on the CALLERLAB Sustainable Square Dancing program.

All of the controls on the Dance screen now function. Select the dance level, continuation (resolve or keep going?), maximum difficulty, and running average difficulty. This may not sound huge, but it also lays the groundwork for handling of more module selection criteria to come. I'm thrilled. üòÖ

Some more caveats:

  • Some features, including half the controls on the Dance page, Are just placeholders. They don't do anything yet.
  • Yes, it is mobile friendly! I wouldn't create modules on a phone, but the Dance page certainly works great on small screens.

  • SqAC is being developed on Google Chrome. It should work with the current versions of Safari, Firefox, Opera, Microsoft Edge, and maybe Internet Explorer 11, but they are not tested at this point.
